# Решение проблемы: запускается старая сборка

## Проблема

При запуске `./bin/CarRental` открывается старая версия приложения.

## Решения

### Вариант 1: Полная пересборка (РЕКОМЕНДУЕТСЯ)

Выполните следующие команды для полной очистки и пересборки:

```bash
cd /home/vladius/Desktop/apps/CarRENTAL/build

# Удаляем старую сборку
rm -rf CMakeFiles CarRental_autogen *.cmake Makefile CMakeCache.txt bin/CarRental

# Пересобираем
cmake ..
cmake --build . -j$(nproc)

# Запускаем
./bin/CarRental
```

### Вариант 2: Если нужно сбросить базу данных

Если база данных уже существует и содержит старые данные (например, только 8 автомобилей вместо 50), удалите её:

```bash
# Бэкап текущей базы (опционально)
cp ~/.carrental/app.db ~/.carrental/app.db.backup

# Удаление базы данных
rm ~/.carrental/app.db

# При следующем запуске база создастся заново с 50 автомобилями
cd /home/vladius/Desktop/apps/CarRENTAL/build
./bin/CarRental
```

### Вариант 3: Быстрая пересборка

```bash
cd /home/vladius/Desktop/apps/CarRENTAL/build
cmake ..
cmake --build . --clean-first -j$(nproc)
./bin/CarRental
```

## Проверка что сборка свежая

Проверьте дату изменения файла:

```bash
ls -lh /home/vladius/Desktop/apps/CarRENTAL/build/bin/CarRental
```

Файл должен иметь свежую дату (только что скомпилирован).

## Проверка количества автомобилей в базе

```bash
sqlite3 ~/.carrental/app.db "SELECT COUNT(*) FROM cars;"
```

Должно быть 50 автомобилей после первого запуска новой версии.
